# 게임 정책 결정

## 요약

프로토타입 게임 플레이·방·UX 정책 6건 확정.

## 결정 목록

### D1 — 추리 실패 패널티

- **결정**: 서버가 자신의 숨김 타일 1장을 **자동 공개**
- **근거**: 턴 지연 최소화, 구현 단순화
- **거부안**: 플레이어가 패널티 타일 직접 선택

### D2 — 탈락 후 행동

- **결정**: 탈락 플레이어는 **관전 모드** — 게임 종료까지 **대기** 또는 **방 퇴장** 가능
- **근거**: 다른 플레이어 경기 관전·채팅 유지, 이탈 자유 보장
- **구현**: `PlayerBoard.spectator = true`, 턴·추리 UI 비활성, 채팅·보드 열람 유지

### D3 — 최대 인원

- **결정**: 방 최대 **4인 고정** (설정 UI 없음)
- **근거**: 보드게임 정식 인원과 일치, 프로토타입 범위 제한

### D4 — 조커 배치

- **결정**: 타일 배분 후 조커 보유 플레이어가 **대표 숫자(0~11) 선택 + 위치 배치**
- **근거**: 사용자 확인 룰 (FirstPlan §2.3)
- **구현**: `game:placeJoker` 이벤트, 모든 플레이어 배치 완료 후 본게임 시작

### D5 — 게임 종료 후

- **결정**: 방 **대기 상태**로 복귀, 방장이 **재시작** 가능
- **근거**: 같은 방에서 연속 플레이, 로비 강제 복귀 없음

### D6 — 모바일 UX

- **결정**: **모바일웹 지원**, **세로(portrait) 보기 기본**, 터치·safe-area 최적화
- **근거**: 모바일에서 세로 모드로 편하게 플레이 (사용자 요구)
- **구현**: `PortraitGameLayout`, `MobileActionBar`, bottom sheet 추리 UI
